# Вопросы для собеседования по iGaming Domain

## Что такое iGaming

<details>
<summary><b>Что вы знаете об iGaming?</b></summary>

iGaming — это индустрия онлайн-гемблинга, включающая: казино, слоты, live casino, ставки на спорт и покерные платформы. Они работают в реальном времени под строгим регулированием.

С точки зрения frontend: высокие требования к производительности, обработка real-time данных, mobile-first UX, фокус на безопасности и функции ответственной игры.

**Основные продукты:**
- **Online Casino**: Слоты, RNG-игры (рулетка, блэкджек), механики джекпотов
- **Live Casino**: Настоящие дилеры, видеостриминг, минимальная задержка
- **Sportsbook**: Live-ставки, обновление коэффициентов в реальном времени, функции cashout
</details>

## Технические вызовы

<details>
<summary><b>Какие ключевые технические вызовы во frontend iGaming?</b></summary>

**Real-time требования:**
- WebSockets / Server-Sent Events для live-данных
- Частые обновления UI (коэффициенты, состояние игры)
- Интерфейс должен корректно обрабатывать real-time данные и обновления state

**Высокая нагрузка:**
- Пиковые события (матчи, турниры)
- Тысячи одновременных пользователей
- Оптимизация производительности и рендеринга критична для live-обновления коэффициентов

**Mobile-first:**
- Большинство пользователей на мобильных
- Оптимизация touch UX
- Работа на слабых устройствах
</details>

## Регулирование и Compliance

<details>
<summary><b>Что нужно знать о регулировании iGaming?</b></summary>

**Лицензии:**
- MGA (Malta)
- UKGC (UK Gambling Commission)
- Curacao

Платформы работают под строгим регулированием и лицензированием, что влияет на UX, валидацию и пользовательские flow.

**Responsible Gambling (важно упомянуть):**
- Лимиты депозитов
- Опции самоисключения
- Reality checks (оповещения о времени/тратах)

**KYC / AML:**
- Верификация личности
- Региональные ограничения
- Проверка возраста
</details>

## Frontend-безопасность в iGaming

<details>
<summary><b>Какие соображения безопасности важны для iGaming frontend?</b></summary>

Frontend не хранит деньги, но:
- **XSS критичен** — может украсть сессии, токены
- **Обработка токенов** — безопасное хранение, refresh flows
- **Управление сессиями** — timeouts, concurrent sessions
- **Anti-fraud UX** — обнаружение подозрительной активности

Ключевая фраза: "На frontend минимизация attack surface критична, особенно учитывая финансовые операции."
</details>

## Специфика UX

<details>
<summary><b>Какие UX-паттерны специфичны для iGaming?</b></summary>

- Минимальное время до первого действия (ставка/spin)
- Анимации для вовлечения (но не в ущерб производительности)
- Чёткий feedback для ставок и результатов
- Ошибки должны быть очевидны и немедленны
- Семантическая разметка, оптимизация ассетов, быстрый и стабильный UI под высокой нагрузкой
</details>

## Советы для собеседования

<details>
<summary><b>Что НЕ стоит говорить на собеседованиях в iGaming?</b></summary>

Избегайте:
- "Я ничего не знаю об iGaming"
- "Это просто сайты с играми"
- "Frontend тут не так важен"

Вместо этого покажите, что понимаете: специфику индустрии, технические вызовы, регуляторную среду и как frontend-навыки здесь применяются.
</details>
